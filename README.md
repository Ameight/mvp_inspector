## Установка

### macOS и Linux — одна команда

```bash
curl -fsSL https://raw.githubusercontent.com/Ameight/mvp_inspector/master/install.sh | bash
```

Скрипт определит ОС, скачает последний бинарный релиз и установит `tl-ide` в `~/.local/bin/`.

После установки добавь `~/.local/bin` в PATH (если ещё нет):

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc  # или ~/.zshrc
source ~/.bashrc
```

---

### Windows

1. Открой страницу [последнего релиза](https://github.com/Ameight/mvp_inspector/releases/latest)
2. Скачай `tl-ide-vX.Y.Z-Windows.zip`
3. Распакуй в удобное место, например `C:\tl-ide\`
4. Запусти `tl-ide.exe` из папки `tl-ide\`

---

### Вручную (из исходников)

Требования: Python 3.10+, Git

```bash
git clone https://github.com/Ameight/mvp_inspector.git
cd mvp_inspector

# Linux / macOS
./run.sh

# Windows
run.bat
```

Скрипт создаст `.venv`, установит зависимости и запустит приложение автоматически.

---

## Запуск

```bash
tl-ide
```

Или, если установлено из исходников:

```bash
./run.sh       # Linux / macOS
run.bat        # Windows
```

Приложение откроется в браузере по адресу **http://localhost:8080**

При первом запуске откроется **wizard** — он спросит, куда сохранить настройки и плагины (рекомендуется `~/.tl-ide/`).

## Список функций

### Плагины
- Загрузка плагинов из произвольной папки (настраивается)
- Группировка по категориям в сайдбаре
- Динамические формы: строки, числа, чекбоксы, textarea, select с ручным вводом
- Асинхронный запуск; вывод результата в Markdown
- Копирование результата в буфер обмена
- Удаление плагина из интерфейса (с подтверждением)
- Генератор нового плагина по шаблону

### Marketplace
- Браузер плагинов из подключённых реестров (JSON)
- Поиск по названию, категории, описанию
- Фильтр по маркетплейсу (если подключено несколько)
- Установка плагина в один клик + автоматическая установка зависимостей (`pip`)
- Выбор конкретной версии для установки (включая даунгрейд)
- Обновление плагина до новой версии из реестра
- Проверка минимальной версии приложения для плагина
- SHA-256 проверка целостности после установки
- Индикатор «⚠ изменён» если файл плагина был изменён вручную

### Настройки
- Управление env-переменными для каждого плагина (с маскировкой секретов)
- Добавление, редактирование и удаление маркетплейсов
- Выбор папки с плагинами через нативный диалог или ввод пути вручную
- Встроенный редактор `config.yaml` с валидацией YAML
- Проверка обновлений приложения вручную

### Обновления приложения
- Автоматическая проверка новой версии при каждом запуске
- Баннер с описанием релиза и кнопкой «Обновить»
- Обновление без выхода из приложения
- Кнопка «Перезапустить» сразу после обновления

### Логи
- Просмотр логов всех плагинов и системы в реальном времени
- Фильтрация по уровню: info / warning / error / debug
- Очистка лога одной кнопкой

### Первый запуск
- Wizard выбора расположения настроек и папки плагинов
- Автоматическое создание `config.yaml` из шаблона
- Перезапуск после настройки с применением новых путей

## Как пользоваться

### Плагины

**Запустить плагин:**
1. Выбери плагин из сайдбара слева (сгруппированы по категориям).
2. Заполни поля формы в основной области.
3. Нажми **▶ Запустить**.
4. Результат появится ниже в виде Markdown.
5. Кнопка **📋 Копировать** скопирует результат в буфер обмена.

**Удалить плагин:**
1. Открой плагин в сайдбаре.
2. Нажми иконку 🗑 (корзина) в правом верхнем углу.
3. Подтверди удаление — файл плагина будет удалён безвозвратно.

**Иконки в сайдбаре:**
- `</>` — локальный плагин (создан вручную)
- 🏪 — плагин из маркетплейса
- ⚠ — файл плагина изменён после установки (целостность нарушена)

---

### Marketplace

**Найти и установить плагин:**
1. Нажми кнопку 🏪 в шапке сайдбара.
2. Используй строку поиска для фильтрации по названию, категории или описанию.
3. Если подключено несколько реестров — фильтруй через выпадающий список справа.
4. Нажми **Установить** на нужной карточке.
5. После установки **перезапусти приложение** — плагин появится в сайдбаре.

**Обновить или откатить версию плагина:**
1. Открой Marketplace.
2. На карточке установленного плагина выбери версию из выпадающего списка.
3. Нажми **Установить** — выбранная версия будет загружена и применена.
4. Перезапусти приложение.

**Значки на карточках:**
- ✓ Установлен — плагин уже есть в папке плагинов
- ⚠ Изменён — файл плагина не совпадает с оригиналом из реестра
- `app ≥ X.Y.Z` — плагин требует минимальную версию приложения

---

### Настройки

Открываются кнопкой ⚙ в шапке сайдбара.

**Env-переменные (токены, секреты):**
1. В секции **Переменные окружения** отображаются все плагины, которым нужны секреты.
2. Зелёный кружок — переменная задана, красный — не задана.
3. Нажми **Изменить** рядом с плагином → введи значения → **Сохранить**.
4. Значения сохраняются в файл `.env` и применяются сразу (без перезапуска).

**Маркетплейсы:**
1. В секции **Маркетплейсы** перечислены подключённые реестры.
2. Измени название или URL и нажми 💾 — изменения сохранятся в `config.yaml`.
3. Нажми 🗑 рядом с реестром — он будет удалён.
4. Введи новые название + URL и нажми **+** для добавления.

**Папка плагинов:**
1. В секции **Плагины** отображается текущая папка.
2. Нажми иконку 📁 для выбора через диалог, или введи путь вручную.
3. Нажми 💾 — путь сохранится в `config.yaml`.
4. Перезапусти приложение для применения.

**Редактор config.yaml:**
1. В секции **config.yaml** показан текущий путь к файлу.
2. Отредактируй содержимое прямо в текстовом поле.
3. Нажми **Сохранить config.yaml** — файл проверится на валидность YAML и сохранится.
4. Перезапусти приложение для применения изменений.

---

### Логи

Открываются кнопкой 🐛 в шапке сайдбара.

- Записи показываются в обратном хронологическом порядке (новые сверху).
- Каждая запись содержит: время, уровень (`info` / `warning` / `error` / `debug`), источник и сообщение.
- **Обновить** — перечитать список.
- **Очистить** — удалить все записи из памяти.

---

### Обновления приложения

**Автоматически при запуске:**
- Приложение проверяет последнюю версию на GitHub через 2 секунды после старта.
- Если доступна новая версия — появится уведомление в правом нижнем углу.

**Вручную:**
1. Открой ⚙ Настройки → секция **Обновления**.
2. Нажми **Проверить обновления**.
3. Если найдена новая версия — появится карточка с описанием релиза.
4. Нажми **Обновить до vX.Y.Z** → затем **Перезапустить**.

> ⚠ Обновление из интерфейса работает только при установке из исходников (`git clone`). Для бинарной установки скачай новый релиз вручную или переустанови через `curl`.

## Создание плагина

**Через интерфейс:**
1. Нажми кнопку **+** в шапке сайдбара.
2. Скопируй шаблон кнопкой 📋.
3. Создай файл вручную по инструкции на экране.

**Структура папок:**

```
plugins/
  devops/
    my_plugin/plugin.py      ← категория DevOps
  jira/
    issue_info/plugin.py
  general/
    ip_checker/plugin.py
```

После создания файла **перезапусти приложение** — плагин появится в сайдбаре.

## SDK плагина

Все плагины наследуют `PluginInterface` из `sdk/base_plugin.py`.

### Методы

| Метод | Обязателен | Описание |
|---|---|---|
| `run(inputs) -> str` | ✅ | Основная логика. Возвращает строку (поддерживает Markdown) |
| `get_display_name()` | — | Название в сайдбаре |
| `get_description()` | — | Описание под заголовком |
| `get_category()` | — | Категория для группировки (по умолчанию `General`) |
| `get_config_schema()` | — | Схема полей формы |
| `get_required_env()` | — | Объявление нужных env-переменных |
| `get_config_key()` | — | Ключ в `config.yaml → plugins` |
| `is_enabled()` | — | `False` — плагин не загружается |

### Типы полей формы

| Тип | Описание |
|---|---|
| `string` | Однострочный ввод |
| `textarea` | Многострочный ввод (код, промпты) |
| `int` | Числовой ввод |
| `bool` | Чекбокс |
| `select_or_input` | Выпадающий список + ручной ввод |

### Пример плагина

```python
import os
from sdk.base_plugin import PluginInterface

class MyPlugin(PluginInterface):

    def get_display_name(self) -> str:
        return "My Tool"

    def get_category(self) -> str:
        return "DevOps"

    def get_required_env(self) -> dict:
        return {
            "MY_TOKEN": {
                "label": "API Token",
                "description": "Токен для доступа к сервису",
                "secret": True,
            }
        }

    def get_config_schema(self) -> dict:
        return {
            "query": {"label": "Запрос", "type": "string", "default": ""},
        }

    def run(self, inputs: dict) -> str:
        token = os.getenv("MY_TOKEN")
        query = inputs.get("query", "")
        # ...
        return f"**Результат:** {query}"
```

**Секреты** (токены) — храни в `.env`, читай через `os.getenv("MY_TOKEN")`.  
**Конфиг** (URL, настройки) — в `config.yaml → plugins.<get_config_key()>`, читай через `self.config.get("url")`.

## Запуск через systemd

Позволяет запускать TL IDE как фоновый сервис: автостарт при входе, автоперезапуск при падении, логи через `journalctl`.

> Только Linux. Используется **user-уровень** systemd — root не нужен.  
> Требует установки из исходников.

### Установка

```bash
make install-service
```

Команда генерирует `~/.config/systemd/user/tl-ide.service` с правильными путями, включает и запускает сервис.

### Управление

```bash
make service-status   # статус
make service-logs     # логи в реальном времени (journalctl -f)
make service-stop     # остановить
make service-start    # запустить
make uninstall-service  # удалить сервис полностью
```

### Обновление

```bash
make update                      # git pull + pip install
systemctl --user restart tl-ide  # применить
```

Или прямо из интерфейса: ⚙ Настройки → Обновления → «Обновить» → «Перезапустить».

### Автозапуск при загрузке системы

По умолчанию user-сервисы стартуют только после входа пользователя:

```bash
loginctl enable-linger $USER
```

## Приватный маркетплейс с API-ключом

Для внутрикорпоративных плагинов, которые не должны быть публично доступны.

### Архитектура

```
[TL IDE клиент]  ──  X-API-Key: <ключ>  ──▶  [marketplace_server.py]
                                                      │
                                              проверяет ключ
                                                      │
                                              отдаёт registry.json
                                              и plugin.py файлы
```

### Запуск сервера

```bash
python marketplace_server.py
```

При первом запуске создаётся `marketplace_server.yaml` — отредактируй его:

```yaml
host: 0.0.0.0
port: 9090
plugins_dir: ./plugins    # папка с registry.json и плагинами

api_keys:
  - key: my-secret-key-abc123   # замени на свой ключ
    name: "Team Alpha"
  - key: another-key-xyz789
    name: "Team Beta"
```

Добавь плагины и сгенерируй реестр:
```bash
python publish.py   # генерирует plugins/registry.json
```

> `marketplace_server.yaml` содержит секреты — он в `.gitignore` и не коммитится.

### Подключение в TL IDE

1. ⚙ Настройки → Маркетплейсы → заполни форму:
   - **Название**: My Private Plugins
   - **URL**: `http://your-server:9090/registry.json`
   - **API Key**: `my-secret-key-abc123`
2. Нажми **+**
3. Перейди в 🏪 Marketplace — приватные плагины появятся в списке

## Создание своего маркетплейса

Маркетплейс — это обычный JSON-файл (`registry.json`), доступный по HTTP. Достаточно публичного GitHub-репозитория, сервер не нужен.

### Быстрый старт

1. Создай репозиторий на GitHub (например `your-org/my-plugins`).
2. Добавь файл `registry.json` в корень.
3. Рядом положи папки с плагинами:

```
my-plugins/
├── registry.json
├── devops/
│   └── deploy_checker/
│       └── plugin.py
└── jira/
    └── sprint_report/
        └── plugin.py
```

4. Подключи реестр: ⚙ Настройки → Маркетплейсы → введи название и URL к `registry.json` → **+**.

URL для GitHub:
```
https://raw.githubusercontent.com/your-org/my-plugins/master/registry.json
```

---

### Формат registry.json

```json
[
  {
    "id": "devops/deploy_checker",
    "name": "Deploy Checker",
    "category": "DevOps",
    "description": "Проверяет статус деплоя по тегу.",
    "author": "your-name",
    "version": "1.0.0",
    "raw_url": "https://raw.githubusercontent.com/your-org/my-plugins/master/devops/deploy_checker/plugin.py",
    "requires": ["requests"],
    "min_app_version": "1.2.0"
  },
  {
    "id": "jira/sprint_report",
    "name": "Sprint Report",
    "category": "Jira",
    "description": "Сводка по спринту: закрыто/в работе/заблокировано.",
    "author": "your-name",
    "version": "2.1.0",
    "raw_url": "https://raw.githubusercontent.com/your-org/my-plugins/master/jira/sprint_report/plugin.py",
    "requires": ["requests", "tabulate"],
    "versions": [
      {
        "version": "2.1.0",
        "raw_url": "https://raw.githubusercontent.com/your-org/my-plugins/master/jira/sprint_report/plugin.py"
      },
      {
        "version": "2.0.0",
        "raw_url": "https://raw.githubusercontent.com/your-org/my-plugins/v2.0.0/jira/sprint_report/plugin.py"
      }
    ]
  }
]
```

### Описание полей

| Поле | Обязателен | Описание |
|---|---|---|
| `id` | ✅ | Уникальный идентификатор. Путь установки: `<plugins_dir>/<id>/plugin.py` |
| `name` | ✅ | Название плагина |
| `category` | ✅ | Категория для группировки |
| `description` | — | Краткое описание |
| `author` | — | Автор |
| `version` | ✅ | Текущая версия `X.Y.Z` |
| `raw_url` | ✅ | Прямая ссылка на `plugin.py` |
| `requires` | — | pip-пакеты — устанавливаются автоматически |
| `min_app_version` | — | Минимальная версия TL IDE |
| `versions` | — | Массив версий для селектора версий |

### Чеклист перед публикацией

- [ ] `id` уникален и совпадает с папкой в репозитории
- [ ] `raw_url` ведёт напрямую на `plugin.py` (проверь в браузере — должен быть сырой Python, не HTML)
- [ ] `version` обновлена
- [ ] Все внешние зависимости перечислены в `requires`
- [ ] Секреты объявлены в `get_required_env()` с понятным `label`
- [ ] Плагин обрабатывает отсутствие токена и возвращает понятное сообщение

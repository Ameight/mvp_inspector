## Требования

- Python 3.10+
- Git

## Установка

```bash
git clone https://github.com/Ameight/mvp_inspector.git
cd mvp_inspector
make install
```

Или без Make:

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск

```bash
make run
# или
python main.py
```

Приложение откроется в браузере по адресу **http://localhost:8080**

При первом запуске откроется **wizard** — он спросит, куда сохранить настройки и плагины (рекомендуется `~/.tl-ide/`).

## Список функций

### Плагины
- Загрузка плагинов из произвольной папки (настраивается)
- Группировка по категориям в сайдбаре
- Динамические формы: строки, числа, чекбоксы, textarea, select с ручным вводом
- Асинхронный запуск; вывод результата в Markdown
- Копирование результата в буфер обмена
- Удаление плагина из интерфейса (с подтверждением)
- Генератор нового плагина по шаблону (`make plugin`)

### Marketplace
- Браузер плагинов из подключённых реестров (JSON)
- Поиск по названию, категории, описанию
- Фильтр по маркетплейсу (если подключено несколько)
- Установка плагина в один клик + автоматическая установка зависимостей (`pip`)
- Выбор конкретной версии для установки (включая даунгрейд)
- Обновление плагина до новой версии из реестра
- Проверка минимальной версии приложения для плагина
- SHA-256 проверка целостности после установки
- Индикатор «⚠ изменён» если файл плагина был изменён вручную

### Настройки
- Управление env-переменными для каждого плагина (с маскировкой секретов)
- Добавление, редактирование и удаление маркетплейсов
- Выбор папки с плагинами через нативный диалог или ввод пути вручную
- Встроенный редактор `config.yaml` с валидацией YAML
- Проверка обновлений приложения вручную

### Обновления приложения
- Автоматическая проверка новой версии при каждом запуске
- Баннер с описанием релиза и кнопкой «Обновить»
- Кнопка закрытия баннера (не удаляет, просто скрывает до перезапуска)
- Обновление через `git fetch + checkout` без выхода из приложения
- Кнопка «Перезапустить» сразу после обновления
- Сохранение `config.yaml` при обновлении (не перезаписывается)

### Логи
- Просмотр логов всех плагинов и системы в реальном времени
- Фильтрация по уровню: info / warning / error / debug
- Очистка лога одной кнопкой

### Первый запуск
- Wizard выбора расположения настроек и папки плагинов
- Автоматическое создание `config.yaml` из шаблона
- Перезапуск после настройки с применением новых путей

## Как пользоваться

### Плагины

**Запустить плагин:**
1. Выбери плагин из сайдбара слева (сгруппированы по категориям).
2. Заполни поля формы в основной области.
3. Нажми **▶ Запустить**.
4. Результат появится ниже в виде Markdown.
5. Кнопка **📋 Копировать** скопирует результат в буфер обмена.

**Удалить плагин:**
1. Открой плагин в сайдбаре.
2. Нажми иконку 🗑 (корзина) в правом верхнем углу.
3. Подтверди удаление в диалоге — файл плагина будет удалён безвозвратно.

**Иконки в сайдбаре:**
- `</> ` — локальный плагин (создан вручную)
- 🏪 — плагин из маркетплейса (наведи — увидишь название реестра)
- ⚠ — файл плагина изменён после установки (целостность нарушена)

---

### Marketplace

**Найти и установить плагин:**
1. Нажми кнопку 🏪 в шапке сайдбара.
2. Используй строку поиска для фильтрации по названию, категории или описанию.
3. Если подключено несколько реестров — фильтруй по маркетплейсу через выпадающий список справа.
4. Нажми **Установить** на нужной карточке.
5. После установки **перезапусти приложение** — плагин появится в сайдбаре.

**Обновить или откатить версию плагина:**
1. Открой Marketplace.
2. На карточке установленного плагина выбери версию из выпадающего списка.
3. Нажми **Установить** — выбранная версия будет загружена и применена.
4. Перезапусти приложение.

**Значки на карточках:**
- ✓ Установлен — плагин уже есть в папке плагинов
- ⚠ Изменён — файл плагина не совпадает с оригиналом из реестра
- `app ≥ X.Y.Z` — плагин требует минимальную версию приложения

---

### Настройки

Открываются кнопкой ⚙ в шапке сайдбара.

**Env-переменные (токены, секреты):**
1. В секции **Переменные окружения** отображаются все плагины, которым нужны секреты.
2. Зелёный кружок — переменная задана, красный — не задана.
3. Нажми **Изменить** рядом с плагином → введи значения → **Сохранить**.
4. Значения сохраняются в файл `.env` и применяются сразу (без перезапуска).

**Маркетплейсы:**
1. В секции **Маркетплейсы** перечислены подключённые реестры.
2. Измени название или URL и нажми 💾 — изменения сохранятся в `config.yaml`.
3. Нажми 🗑 рядом с реестром — он будет удалён.
4. В нижней карточке введи новые название + URL и нажми **+** для добавления.

**Папка плагинов:**
1. В секции **Плагины** отображается текущая папка.
2. Нажми иконку 📁 для выбора через диалог, или введи путь вручную.
3. Нажми 💾 — путь сохранится в `config.yaml`.
4. Перезапусти приложение для применения.

**Редактор config.yaml:**
1. В секции **config.yaml** показан текущий путь к файлу и его источник.
2. Отредактируй содержимое прямо в текстовом поле.
3. Нажми **Сохранить config.yaml** — файл проверится на валидность YAML и сохранится.
4. Перезапусти приложение для применения изменений.

---

### Логи

Открываются кнопкой 🐛 в шапке сайдбара.

- Записи показываются в обратном хронологическом порядке (новые сверху).
- Каждая запись содержит: время, уровень (`info` / `warning` / `error` / `debug`), источник и сообщение.
- **Обновить** — перечитать список (если логи появились только что).
- **Очистить** — удалить все записи из памяти (не влияет на файл лога).

---

### Обновления приложения

**Автоматически при запуске:**
- Приложение проверяет последнюю версию на GitHub через 2 секунды после старта.
- Если доступна новая версия — появится уведомление в правом нижнем углу.
- Кнопка **✕** закрывает уведомление (до следующего запуска).

**Вручную:**
1. Открой ⚙ Настройки → секция **Обновления**.
2. Нажми **Проверить обновления**.
3. Если найдена новая версия — появится карточка с описанием релиза.
4. Нажми **Обновить до vX.Y.Z** — приложение выполнит `git fetch + checkout` в фоне.
5. После завершения нажми **Перезапустить**.

> ⚠ Перед обновлением зафиксируй локальные изменения (`git stash` или `git commit`) — иначе обновление будет заблокировано.

## Создание плагина

**Через генератор (рекомендуется):**

```bash
make plugin name=my_plugin category=devops
# или
python create_plugin.py my_plugin devops
```

Создаётся файл `plugins/devops/my_plugin/plugin.py` с готовым шаблоном.

**Через интерфейс:**
1. Нажми кнопку **+** в шапке сайдбара.
2. Скопируй шаблон кнопкой 📋.
3. Создай файл вручную по инструкции на экране.

**Структура папок плагинов:**

```
plugins/
  devops/
    my_plugin/plugin.py      ← категория DevOps
  jira/
    issue_info/plugin.py     ← категория Jira
    my_tasks/plugin.py
  general/
    ip_checker/plugin.py
```

После создания файла **перезапусти приложение** — плагин появится в сайдбаре.

## SDK плагина

Все плагины наследуют `PluginInterface` из `sdk/base_plugin.py`.

### Методы

| Метод | Обязателен | Описание |
|---|---|---|
| `run(inputs) -> str` | ✅ | Основная логика. Возвращает строку (поддерживает Markdown) |
| `get_display_name()` | — | Название в сайдбаре |
| `get_description()` | — | Описание под заголовком |
| `get_category()` | — | Категория для группировки (по умолчанию `General`) |
| `get_config_schema()` | — | Схема полей формы |
| `get_required_env()` | — | Объявление нужных env-переменных |
| `get_config_key()` | — | Ключ в `config.yaml → plugins` (по умолчанию snake_case имени класса) |
| `is_enabled()` | — | `False` — плагин не загружается |

### Типы полей формы

| Тип | Описание |
|---|---|
| `string` | Однострочный ввод |
| `textarea` | Многострочный ввод (код, промпты) |
| `int` | Числовой ввод |
| `bool` | Чекбокс |
| `select_or_input` | Выпадающий список + ручной ввод |

### Пример плагина

```python
import os
from sdk.base_plugin import PluginInterface

class MyPlugin(PluginInterface):

    def get_display_name(self) -> str:
        return "My Tool"

    def get_category(self) -> str:
        return "DevOps"

    def get_required_env(self) -> dict:
        return {
            "MY_TOKEN": {
                "label": "API Token",
                "description": "Токен для доступа к сервису",
                "secret": True,
            }
        }

    def get_config_schema(self) -> dict:
        return {
            "query": {"label": "Запрос", "type": "string", "default": ""},
        }

    def run(self, inputs: dict) -> str:
        token = os.getenv("MY_TOKEN")
        query = inputs.get("query", "")
        # ...
        return f"**Результат:** {query}"
```

**Секреты** (токены) — храни в `.env`, читай через `os.getenv("MY_TOKEN")`.  
**Конфиг** (URL, настройки) — в `config.yaml` → `plugins.<get_config_key()>`, читай через `self.config.get("base_url")`.

## Структура проекта

```
mvp_inspector/
├── main.py               # точка входа и весь UI
├── create_plugin.py      # генератор плагинов
├── Makefile
├── requirements.txt
├── config.example.yaml   # шаблон конфига
├── config.yaml           # активный конфиг (не коммитить если содержит секреты)
├── .env                  # секреты (токены) — не коммитить
├── sdk/
│   └── base_plugin.py    # базовый класс PluginInterface
└── plugins/              # папка плагинов (путь настраивается)
    ├── manifest.json     # реестр установленных плагинов
    └── <категория>/
        └── <плагин>/
            └── plugin.py
```
